# Patch Mining — Data Directory

All datasets used in the SCORED 2026 paper:
*"From Fix Commits to Vulnerable Functions: Automated Patch Mining for Function-Level CVE Resolution"*

## Files

### `ghsa_js_python_extraction.csv` (162 entries)
- **Source:** GHSA pipeline run on npm + PyPI advisories
- **Origin:** Research data repository exports
- **Ecosystems:** npm (90), PyPI (72)
- **Key stats:** 127 fetchable diffs, 95 with extracted functions, 328 total functions
- **Used in:** Table 4 (extraction rates by ecosystem), cross-ecosystem comparison

### `ghsa_study1_precision_corpus.csv` (45 entries)
- **Source:** Curated subset of the above for manual precision validation
- **Origin:** Research data repository
- **Ecosystems:** npm (23), PyPI (22)
- **Relationship:** Strict subset of `ghsa_js_python_extraction.csv` (all 45 IDs appear in the 162)
- **Key stats:** 41 with fix commits, 31 with extracted functions, 133 total functions
- **Used in:** Study 1 (RQ1 precision), Tables 2–3

### `project_kb_java_extraction.csv` (1,248 entries)
- **Source:** Pipeline run on SAP Project KB fix commits
- **Origin:** Generated by `evaluation/scored_evaluation.py`
- **Ecosystems:** Java/Maven (primary), with some JS/Python/Go entries from KB
- **Key stats:** 1,109 fetchable diffs, 954 with extracted functions, 8,069 total functions
- **Used in:** Study 2 (RQ2 extraction coverage), Tables 4–5

### `project_kb_validation_summary.txt`
- **Source:** Summary statistics from the Project KB extraction run
- **Generated:** 2026-05-16

## Schema

All CSV files share the same schema:
```
vuln_id, affected_package, ecosystem, severity, fix_commits_found,
functions_mined, num_functions_mined, functions_in_call_graph,
num_in_call_graph, reachability_verdicts, has_unreachable, source_repo
```

The Project KB CSV uses a different schema:
```
vuln_id, repo, packages, num_commits_kb, commits_processed,
functions_extracted, num_functions
```

## Reproducibility

The extraction pipeline source is documented in `src/README.md`.
Evaluation scripts and precision validation data are in `evaluation/`:

- `evaluation/precision_validation_samples.csv` — 50 JS/Python precision verdicts (Central/Tangential/Noise)
- `evaluation/java_precision_validation.csv` — 20 Java cross-ecosystem precision verdicts
- `evaluation/scored_evaluation.py` — Project KB extraction and evaluation
- `evaluation/generate_validation_samples.py` — Stratified sampling for precision study
- `evaluation/verify_verdicts.py` — Automated verdict verification against commit diffs

To reproduce the evaluation summary:
```bash
python evaluation/scored_evaluation.py
```
